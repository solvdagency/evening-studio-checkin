<objective>
Build the delivery transport: a single non-throwing `postToChat(payload, webhookUrl)` that POSTs the Cards v2 JSON to the Google Chat incoming webhook and returns a `Result`-style value. This is the side-effecting twin of the pure renderer — isolated so the renderer stays pure and the post is independently mockable. It mirrors the existing `src/productive/client.ts` boundary pattern exactly.

Purpose: REL-02 hangs off this — a failed post must surface as `{ ok: false }` so the composition root (plan 04) can exit non-zero and fire the GitHub failure email. The webhook URL is a secret and must never leak into logs/errors.
Output: `src/chat/postToChat.ts` + its stubbed-fetch test.
</objective>
