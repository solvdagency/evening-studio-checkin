  export interface StudioReportInput {
    now: DateTime;
    holidays: HolidaySet;
    roster: ReadonlyArray<DesignerId>;
    bookings: ReadonlyArray<DatedBooking>;       // DatedBooking extends Booking with optional `date` ("yyyy-MM-dd")
    absences: ReadonlyArray<DatedAbsence>;
    assessedDesigners?: ReadonlyArray<DesignerId>; // OMIT = whole roster assessed; pass ONLY designers actually reached
  }
  export function computeStudioReport(input: StudioReportInput): StudioReport;

CRITICAL assessedDesigners semantics (report.ts lines 76-87): present-but-empty (designer in the list, zero bookings) = underbooked with full open; absent-from-list = missingDesigners (silent-partial-result guard, T-01-06). gather MUST pass only designers it actually reached.

Window: target→Friday. gather derives the target day the same way the report does. To attribute window-day minutes for the rollup, gather sets the `date` field on DatedBooking/DatedAbsence per window day (mappers.minutesOnDay already takes a dayKey). The brief checks apply ONLY to target-day bookings (D-08).

Composition root style to mirror (src/domain/report.ts computeStudioReport lines 164-223): destructure an input object, call pure helpers in order, return a well-formed object even on empty/partial data.
