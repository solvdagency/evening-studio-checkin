<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| run-log object → committed `.runs/*.json` (then pushed to the public-ish repo) | A serialised run log crosses into version control; any secret embedded here is durably leaked |
| postToChat error string → run-log postOutcome | A redacted-or-not reason string crosses from the POST layer into the persisted log |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | buildRunLog → `.runs/*.json` contents | mitigate | RunLog carries only counts, booleans, a date string, an enum, and an already-redacted reason; Task 1 asserts no webhook fragment / "GOOGLE_SA_KEY" / "sk-ant-" survives JSON.stringify (D-08) |
| T-07-02 | Tampering | markerDateKey clock source | mitigate | Date key derives ONLY from the injected `now` via STUDIO_ZONE; grep gate forbids `DateTime.now(` in marker.ts (D-03, preserves the single-clock trust boundary) |
| T-07-03 | Denial of Service (self-inflicted missed night) | writeMarker throwing | mitigate | writeMarker returns a Result and never throws, so Plan 02 can log-and-exit-0 on a post-success marker-write failure (D-07-fail) rather than firing a false failure alert |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies introduced (luxon + node:fs only, both already present); slopcheck N/A this plan |
