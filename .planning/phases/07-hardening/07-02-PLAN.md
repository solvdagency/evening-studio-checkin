<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| postToChat result → run-log postOutcome → committed `.runs/*.json` | A POST error string could carry the webhook URL; it must be redacted before it is persisted to version control |
| GitHub Actions GITHUB_TOKEN → repo write | `contents: write` widens the default token scope; a marker commit must not be able to trigger an infinite workflow loop |
| `process.env.GITHUB_EVENT_NAME` → idempotency decision | The scheduled-vs-manual branch trusts this env var to decide whether the guard engages |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Elevation of Privilege | nightly.yml `contents: write` | mitigate | Permission declared at JOB level on `checkin` only (not workflow-wide), keeping the GITHUB_TOKEN scope minimal; no other permission granted |
| T-07-05 | Tampering (runaway automation) | marker commit re-triggering the workflow | mitigate | The marker commit message carries `[skip ci]`; the schedule/dispatch triggers are untouched, so the commit cannot start a new run |
| T-07-06 | Information Disclosure | postOutcome reason in `.runs/*.json` | mitigate | runNightly never passes the webhook URL or any secret to buildRunLog/console.*; buildRunLog stores only the already-redacted reason (inherits Plan 01 T-07-01 redaction; Task 2 asserts posted-path facts carry no secret) |
| T-07-07 | Denial of Service (false failure alert) | writeMarker failure after a good post | mitigate | D-07-fail: a failed marker write logs a loud warning and returns 0, so a persist hiccup never fires GitHub's failed-run email and never masks a real success |
| T-07-08 | Spoofing | trusting GITHUB_EVENT_NAME | accept | The env var is set by the GitHub Actions runtime, not user input; a forged value only changes whether the idempotency guard engages, whose worst case is a self-inflicted double-post to the test space (D-04 accepted risk) |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No new dependencies (uses Plan 01's module + git CLI already on the runner); slopcheck N/A this plan |
</threat_model>
