never alter a figure — via a number-fidelity property test.

and fallback-integrity tests + the LLM dep wired behind a flag in runNightly + a structured
run-log line.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model output → card | Untrusted LLM text (headerSentence, verdict lines) crosses into the Cards v2 HTML-subset payload. |
| env → process | ANTHROPIC_API_KEY (secret) enters at the client boundary; must never reach a log/card. |
| npm registry → repo | A new runtime dependency (@anthropic-ai/sdk) is installed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | model output → numeric fields | mitigate | Cardinal guardrail: assembleCardsV2 sources every figure/row/dot/link from report/ctx; model text inserted as prose only and never parsed for numbers. Proven by the numberFidelity property test (Task 1/2). |
| T-05-02 | Information Disclosure | console logs | mitigate | Run-log + warn lines carry only failure class/tokens/cost; never the key or webhook URL. Asserted in Task 3 (grep for sk-ant/key/webhook host). |
| T-05-03 | Tampering | model prose → Cards v2 HTML | mitigate | escapeHtml every model string (headerSentence, line) before insertion, reusing the existing renderer primitive. |
| T-05-04 | Denial of Service | nightly Anthropic call | mitigate | client timeout 20_000 + max_tokens 1024 + maxRetries 2; truncation/timeout/refusal → fallback to template (REL-01), never a hung or missed post. |
| T-05-SC | Tampering | npm install @anthropic-ai/sdk | mitigate | First-party Anthropic SDK — see RESEARCH Package Legitimacy gate / Task 0a checkpoint above before install. |
</threat_model>
